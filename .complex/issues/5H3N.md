Decouple ox-server from cx. External event ingestion becomes a plugin boundary: standalone watcher processes push source events to `POST /api/events/ingest`; ox-server has no built-in knowledge of cx, GitHub, Linear, or any other tracker.

See:
- `docs/prd/event-sources.md` — PRD
- `docs/event-sources-design.md` — implementation design

Five vertical slices, each red/green, each ships. Order: PRD update → design doc update → implementation.

1. Ingest endpoint + SourceEvent type (server accepts batches; nothing calls it)
2. Generic trigger matching on source events
3. `ox-cx-watcher` crate in parallel to in-server poller
4. Switchover: `ox-ctl up` launches the watcher, cx poller disabled
5. Delete the legacy path and scrub remaining stale docs

The engine below the ingest point is already source-agnostic. All change lives above the event-bus append call.
