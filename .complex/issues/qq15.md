## Motivation

After tXwK shipped, a non-retriable signal (e.g. `auth_failed`) escalates the execution on the first failure. Recovery is not one command away: `escalated` executions are terminal, and the trigger-dedup at `ox-herder/src/herder.rs:984-989` blocks new executions for the same `(origin, workflow)` while any prior execution is `running` OR `escalated`. So even after fixing auth, re-emitting `cx node.ready` won't fire — dedup suppresses it.

This blocks the actual user flow: rotate creds → retry the workflow.

## Design

New `ox-ctl exec retry <id> [--from-start]`:

- Default behavior: restart from the **failed step**. The failed step is whatever `current_step` was when escalation projected (always set by step.dispatched). Workspace continuity is automatic — the per-task git branch already holds prior steps' commits.
- `--from-start` flag: ignore the failed-step hint, start from `workflow.first_step()`.
- Scope: only escalated executions. `running`/`awaiting_step` retries are a different problem (need to kill in-flight runners cleanly first) — out of scope here.

## Mechanics

Server-side retry endpoint `POST /api/executions/{id}/retry?from-start=<bool>`:

1. Look up `ExecutionState` for `<id>` (404 if not found, 409 if status != Escalated).
2. Capture workflow / vars / origin / current_step from the projection.
3. Emit `execution.cancelled` for `<id>` (releases the dedup lock).
4. Emit `execution.created` for a freshly-minted exec_id, with `start_step = (if from_start { None } else { Some(current_step) })`.
5. Respond with the new execution_id.

`ExecutionCreatedData` gains `start_step: Option<String>` (additive, `#[serde(default)]`). Herder honors it when picking the initial `Ready` phase: falls back to `workflow.first_step()` when None (existing behavior).

Client method `retry_execution(id, from_start) -> Result<ExecutionId>`. ox-ctl subcommand wraps it.

## Slices (red→green within one cx node)

1. `start_step` plumbing through `ExecutionCreatedData` + herder honors it
2. Server `/retry` endpoint
3. Client method + ox-ctl subcommand (trivial wrappers)
