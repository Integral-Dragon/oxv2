ox

Deterministic workflows for hands-off agentic work.

Agents propose, write, review, and investigate. Ox triggers workflows, dispatches steps, records artifacts, retries failures, and escalates when the facts say to stop.

the thesis

Agents should do judgment-heavy work. Coordination should be deterministic.

Ox does not ask an agent what happens next. Workflows advance from explicit events: a task became ready, a step was confirmed, a runner missed heartbeat, a merge failed, or a signal was observed.

Agent outputs are data. Workflow transitions are policy.

the loop

  1. event — an external source says work is ready.
  2. dispatch — ox assigns the next workflow step to an isolated runner.
  3. execute — an agent or human process works in a fresh branch workspace.
  4. confirm — the runner collects artifacts, signals, metrics, pushes, and confirms.
  5. advance — deterministic transitions decide whether to continue, retry, merge, or escalate.

the boundary

agents do cognition

plans, code, reviews, investigations, summaries, tradeoffs, clarifying questions.

ox does coordination

triggers, state, dispatch, retries, timeouts, artifacts, branch discipline, escalation.

three components

ox-server

the hub. owns shared state, exposes HTTP + SSE, the single writer to main. passive — only responds to requests.

ox-herder

the active loop. reads from ox-server, counts things, fires triggers. not AI. dumb on purpose.

ox-runner

the executor. stateless and fungible. runs anywhere — seguro VM, cloud, k8s, bare metal. registers and waits.

principles

  • deterministic orchestration. agents produce outputs and artifacts. workflows decide what those facts mean.
  • events are facts. all state is a projection of an append-only log. history is always recoverable.
  • isolated runners. each step gets a fresh workspace and runtime process.
  • two‑phase completion. a step isn't done until the runner confirms it. the herder never races in-flight work.
  • artifacts are first-class. logs, commits, cx activity, metrics, and declared files are addressable outputs.
  • single writer to main. nothing writes to main except ox-server's merge. the merge is the serialisation point.
  • humans review escalations. people set direction and handle exceptions instead of sitting in the happy path.
  • agents never coordinate agents. no AI in the infrastructure layer. intelligence lives inside bounded steps.
